## 3. 데이터 명세

### 3-1. 핵심 데이터

**이중 데이터 레이어 구조:**

| 레이어 | 소스 | 용도 |
|--------|------|------|
| 정적 연간 데이터 | `tradeData.generated.ts` (~394K LOC) | 연도별 집계, 국가 랭킹, 품목 트리맵, 시계열 |
| 동적 월별 데이터 | Supabase RPC (trade_mti6 테이블) | 월 선택 시 KPI·지도·트리맵 실시간 조회 |

**정적 데이터 구조 (lib/data.ts):**

| 데이터 | 설명 |
|--------|------|
| `KPI_BY_YEAR` | 연도별 전체 수출/수입/무역수지 + 전년 대비 증감률 |
| `COUNTRY_DATA` | 연도×무역유형별 국가 목록 (순위, 금액, 지역, 주요품목) |
| `TREEMAP_DATA` | 연도×무역유형별 MTI 6단위 품목 (코드, 명칭, 금액, 색상) |
| `TIMESERIES_BY_YEAR_COUNTRY` | 연도×국가별 월별 수출/수입/무역수지 |
| `PRODUCT_EXP/IMP_TREND_BY_CODE` | 품목별 연도 추이 |
| `PRODUCT_EXP/IMP_TOP_COUNTRIES_BY_CODE` | 품목별 상위 교역국 |

**Supabase 테이블:**

| 테이블 | 설명 |
|--------|------|
| `trade_mti6` | 월별 무역 원데이터 (YYMM 기준) |
| `chat_logs` | 챗봇 대화 기록 (user_id, role, content, created_at) |
| `auth.users` | Supabase 내장 인증 테이블 |

**Supabase RPC 함수:**

| RPC | 파라미터 | 반환 |
|-----|---------|------|
| `get_treemap_mti6` | p_yymm, p_mode, p_mti_depth | MTI 코드별 금액 (트리맵용) |
| `get_country_map_mti6` | p_yymm, p_mode | 국가별 순위·금액 (지도용) |
| `get_country_treemap_mti6` | p_yymm, p_ctr_name, p_mode, p_mti_depth | 특정 국가 품목 구성 |

### 3-2. MTI 품목 분류 체계

```
MTI 1단위 (대분류, 10개)
 └ MTI 2단위 (중분류, 49개)
   └ MTI 3단위 (소분류, 198개)
     └ MTI 4단위
       └ MTI 5단위
         └ MTI 6단위 ← DB 저장 기준 + UI 기본 표시 단위
```

> **구현 완료:** FilterBar에서 MTI 1~6단위 깊이 선택 가능. `aggregateTreemapByDepth(data, depth)` 함수로 6단위 데이터를 N단위로 실시간 집계.

### 3-3. 데이터 품질 주의사항

- **IMP_AMT = 0인 행 42.3%:** 수입 집계 쿼리에 0값 제외 처리
- **수출 FOB / 수입 CIF:** 동일 품목도 수입 금액이 높게 나올 수 있음
- **2026년 데이터:** 1~2월만 존재 → "불완전 연도" 감지 로직 구현 완료 (KPIBar, RechartsTooltip)
- **홍콩:** 중계무역 허브 → LLM 시스템 프롬프트에 명시
- **라이베리아·마셜제도:** 선박 등록지 → LLM 시스템 프롬프트에 명시

### 8-5. 서브탭 2 — 시계열 추이

- 정적 데이터(`TIMESERIES_BY_YEAR_COUNTRY`)에서 조회
- 연도 선택 가능

### 8-6. 국가별 KPI (getCountryKpi)

- 해당 국가 수출액/수입액/무역수지 + 전년 대비 증감률

## 10. LLM 챗봇 연동

### 10-1. 아키텍처

```
[ChatBot.tsx] → POST /api/chat → [Anthropic SDK] → Claude Haiku 4.5
     ↑                                    │
     │         스트리밍 (ReadableStream)    │
     └────────────────────────────────────┘
```

- **모델:** Claude Haiku 4.5 (`claude-haiku-4-5-20251001`)
- **SDK:** @anthropic-ai/sdk v0.88.0
- **스트리밍:** ReadableStream + TextEncoder (글자 단위 실시간 표시)

### 10-2. 시스템 프롬프트 (고정)

- 한국 무역통계 분석 어시스턴트 역할
- 정확한 데이터 인용 강조
- `==주제==` 하이라이트 포맷 사용 (응답당 1~2개)
- 거시경제 지표 연계 해석 안내
- FOB/CIF, 홍콩 중계무역, 선박 등록지 도메인 지식

### 10-3. 동적 컨텍스트 인젝션 (buildChatContext)

매 질의 시 사용자 메시지를 분석하여 관련 데이터 자동 주입:

| 감지 항목 | 처리 |
|----------|------|
| 국가명 | COUNTRY_LIST에서 매칭 → 해당 국가 KPI·순위 주입 |
| 품목명 | TREEMAP에서 매칭 → 해당 품목 추이·상위국 주입 |
| MTI 코드 | VALID_CODE_PREFIXES 검증 → 코드 기반 조회 |
| 연도 | 2020~2026 추출 → 해당 연도 데이터 주입 |

### 10-4. 네비게이션 버튼 (resolveRouteButtons)

챗봇 응답에 관련 페이지 바로가기 버튼 자동 생성:
- 국가명 언급 → `/country/[name]` 버튼
- 품목명 언급 → `/product/[name]` 버튼
- "추이" 키워드 → `?tab=timeseries` 파라미터 추가
- "상위 국가" 키워드 → `?tab=countries` 파라미터 추가
- MTI 단위 감지 → `?mtiDepth=N` 파라미터 추가
- 연도 감지 → `?year=YYYY` 파라미터 추가

### 10-5. 채팅 기록

- Supabase `chat_logs` 테이블에 저장 (로그인 사용자만)
- 최대 50개 기록 조회
- `/api/welcome` — 이전 대화 기반 맞춤 환영 메시지 생성

### 10-6. 챗봇 UI 기능

- 마크다운 렌더링 (GFM: 테이블, 취소선, 리스트)
- FAQ 제안 버튼
- 글꼴 크기 조절
- 타이핑 인디케이터
- 접기/펼치기 토글

## 13. 상태 관리

**URL 파라미터 + React useState** 방식 (~~Zustand~~ 미사용):

| 상태 | 관리 방식 |
|------|----------|

> 페이지 간 전역 상태 동기화 없음 — 각 페이지가 독립적으로 URL 파라미터 읽기
