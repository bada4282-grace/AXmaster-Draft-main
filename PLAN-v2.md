> 기준 문서: PRD-v2.0  
> 작성일: 2026-04-13  
> 구현 방식: 페이즈별 순차 진행 + 페이즈 내 트랙 병렬 구현

---

## 읽기 전에

- **페이즈(Phase):** 순서대로 진행. 이전 페이즈 완료 후 다음 시작.
- **트랙(Track):** 같은 페이즈 안에서 동시에 진행 가능한 작업 묶음.
- **완료 처리:** 작업 완료 시 `[ ]` → `[x]`
- **블로커:** ⚠️ 표시된 항목은 외부 결정 또는 팀 확인이 필요한 선행 조건.


## Phase 3 — 전역 상태 — 수출/수입 토글

> 목표: 수출/수입 모드가 전역 상태로 관리되어 지도·트리맵·KPI·탭명·카드 레이블 전체에 자동 반영.

- [ ] 전역 상태 스토어 설계 (Zustand 권장)

```typescript
interface TradeStore {
  mode: 'export' | 'import';         // 수출/수입 토글
  year: number;
  month: number | null;
  period: 'monthly' | 'cumulative' | 'annual';
  country: string | null;            // 선택 국가 (P2-A용)
  tab: 'country' | 'product';        // 랜딩 탭 상태
  setMode: (mode: 'export' | 'import') => void;
  setYear: (year: number) => void;
  setMonth: (month: number | null) => void;
  setPeriod: (period: string) => void;
  setCountry: (country: string | null) => void;
  setTab: (tab: 'country' | 'product') => void;
}
```

- [ ] 수출/수입 토글 변경 → 전역 mode 업데이트
- [ ] mode 변경 시 지도·트리맵·KPI 자동 리렌더 확인
- [ ] URL 파라미터 동기화 (`?mode=export` / `?mode=import`)
- [ ] 탭 전환 라우팅 로직 — 국가별 탭에서 품목별 탭 클릭 시 랜딩 품목별 탭으로 이동 (랜딩 국가별 탭으로 돌아가지 않음)

## Phase 6 — P2-A 국가별 상세페이지

## Phase 7 — P2-B 품목별 상세페이지

- [ ] 좌측 카드: 품목명(동적) + 전월 수출(수입)액(고정) + 증감률(고정)

- [ ] **단계 2 — 중분류 (최하단)**

### Track C — 탭 1: 상세 품목별

- [ ] 단계 1(대분류): 선택 대분류 내 중분류 TOP3 트리맵 + 전체 수출(수입) 내 비중
- [ ] 단계 2(중분류): 같은 대분류 내 형제 중분류 비교 트리맵 (현재 위치 강조)
- [ ] 필터 연동

### Track D — 탭 2: 수출(수입)액 추이

- [ ] 월별 라인 차트
- [ ] **X축: 해당 연도의 1월~12월** / 연도 선택은 차트 내 셀렉터
- [ ] Y축: 자동 스케일링
- [ ] 빈 월: 빈값 처리
- [ ] 현재 드릴다운 단계 기준 집계

### Track E — 탭 3: 상위 수출(수입)국

## 용어 정리

| 용어 | 설명 |
|------|------|
| 페이즈 (Phase) | 개발 단계. 이전 페이즈 완료 후 다음 시작. 순서 있음. |
| 트랙 (Track) | 같은 페이즈 안에서 동시에 진행 가능한 작업 묶음. |
| 의존성 | A 작업이 끝나야 B 작업을 시작할 수 있는 관계. |
| 코로플레스 지도 | 지역별로 수치에 따라 색상 농도를 다르게 표시하는 지도. |
| 복합 인덱스 | 여러 컬럼을 묶어서 만든 DB 인덱스. 필터 조합 쿼리 속도를 높임. |
| 캐싱 테이블 | 자주 쓰는 집계 결과를 미리 계산해 저장해둔 별도 테이블. |
| GeoJSON | 지도 국가 경계선 등 지리 데이터를 JSON 형식으로 저장한 파일. |
| SSR 충돌 | Leaflet은 브라우저 전용이라 Next.js 서버 렌더링과 충돌함. `dynamic import + ssr:false`로 해결. |
| 스트리밍 응답 | LLM 답변을 한꺼번에 받지 않고 글자 단위로 실시간으로 받아 표시하는 방식. |
| FOB | Free On Board. 수출 금액 기준. 선적 시점까지의 비용만 포함. |
| CIF | Cost, Insurance and Freight. 수입 금액 기준. 보험료·운임 포함. 동일 품목이라도 FOB보다 높게 나옴. |
| 전역 상태 | 여러 컴포넌트에서 공유하는 데이터 (수출/수입 모드, 필터 등). Zustand로 관리. |
| 드릴다운 | 큰 카테고리 → 세부 카테고리로 단계적으로 좁혀 들어가는 시각화 인터랙션. |
| E2E 테스트 | 실제 브라우저에서 사용자 동작을 자동으로 시뮬레이션해 전체 흐름을 검증. |
